# Отчет по проекту: Автоматизированная система мониторинга состояния ЛЭП

## 1. Концепция и цель проекта

Система разработана для автоматизации процесса технического осмотра линий электропередачи. Вместо ручного пересмотра тысяч фотографий, сделанных БПЛА, нейросеть берет на себя задачу обнаружения объектов и их дефектов.

* **Цель:** Повышение скорости и точности детекции дефектов изоляторов.
* **Метод:** Компьютерное зрение на базе архитектуры **YOLOv8**.

## 2. Структура проекта

Проект организован для обеспечения чистоты кода и удобства запуска:

* `notebook/notebook.ipynb` — **полное пошаговое описание реализации**. Здесь задокументирован весь процесс: от скачивания и первичного анализа данных до обучения модели и проведения экспериментов.
* `main/processor.py` — основной класс `InsulatorInspector`, отвечающий за логику предсказаний и работу с моделью.
* `main/test.py` — скрипт для пакетной обработки данных.
* `model/best.pt` — файл обученных весов нейросети.
* `test_data/` — рабочая директория для тестирования:
* `input/` — папка для исходных фотографий.
* `output/` — папка с результатами детекции и CSV-отчетом.


* `requirements.txt` — зависимости проекта.

## 3. Исходные данные

В качестве основы использовался публичный датасет:
[InsulatorDataSet](https://github.com/InsulatorData/InsulatorDataSet)

## 4. Процесс обучения модели

*Подробный технический разбор процесса обучения доступен в `notebook/notebook.ipynb`.*

1. **Подготовка данных:** Конвертация аннотаций из формата VOC XML в формат YOLO (`.txt` файлы с координатами).
2. **Разметка:** Разделение датасета на *Train (80%)*, *Val (10%)* и *Test (10%)*.
3. **Конфигурация:** Настройка `data.yaml` для классификации объектов (`0: insulator`, `1: defect`).
4. **Обучение:** Дообучение `yolov8n.pt` на 50 эпохах.
5. **Оценка:** Проверка качества модели на тестовом наборе.

## 5. Метрики качества (Итоговые показатели)

| Метрика | Значение | Что означает |
| --- | --- | --- |
| **Precision** | > 0.90 | Высокая точность, минимум ложных срабатываний. |
| **Recall** | > 0.88 | Высокая полнота, обнаружение почти всех реальных дефектов. |
| **mAP@50** | > 0.90 | Высокая общая точность детекции на снимках. |

## 6. Как работает система

Система состоит из двух логических блоков:

* **Ядро детекции (`processor.py`):** Загружает веса, обрабатывает изображения и сохраняет визуализацию (рамки объектов).
* **Аналитический блок (`test.py`):** Выполняет пакетную обработку папки `input`, генерирует `report.csv` с данными о классах и вероятностях (`confidence`).

## 7. Результаты внедрения

* **Автоматизация:** Полный отказ от ручного просмотра исправных участков ЛЭП.
* **Скорость:** Высокая производительность обработки изображений.
* **Точность:** Надежное выявление дефектов, позволяющее оператору быстро принимать решения.
