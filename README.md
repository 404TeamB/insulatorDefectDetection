# Отчет по проекту: Автоматизированная система мониторинга состояния ЛЭП

## 1. Концепция и цель проекта

Система разработана для автоматизации процесса технического осмотра линий электропередачи. Вместо ручного пересмотра тысяч фотографий, сделанных БПЛА, нейросеть берет на себя задачу обнаружения объектов и их дефектов.

* **Цель:** Повышение скорости и точности детекции дефектов изоляторов.
* **Метод:** Компьютерное зрение на базе архитектуры **YOLOv8**.

## 2. Исходные данные

В качестве основы использовался публичный датасет:
[InsulatorDataSet](https://github.com/InsulatorData/InsulatorDataSet)

## 3. Процесс обучения модели

Обучение проводилось в пять ключевых этапов:

1. **Подготовка данных:** Конвертация аннотаций из формата VOC XML в формат YOLO (файлы `.txt` с координатами объектов).
2. **Разметка:** Разделение датасета на выборки:
* *Train (80%)* — для тренировки.
* *Val (10%)* — для валидации.
* *Test (10%)* — для итоговой проверки.


3. **Конфигурация:** Создание файла `data.yaml`, который указывает модели, где искать данные и какие классы объектов искать: `insulator` (изолятор) и `defect` (дефект).
4. **Обучение:** Дообучение модели `yolov8n.pt` на 50 эпохах. Это позволило модели «выучить» визуальные признаки изоляторов и типов повреждений.
5. **Оценка:** Проверка качества модели на независимом тестовом наборе.

## 4. Метрики качества (Итоговые показатели)

Эффективность модели подтверждена высокими значениями метрик на тестовой выборке:

| Метрика | Значение | Что означает |
| --- | --- | --- |
| **Precision** | > 0.90 | Модель крайне редко принимает сторонние объекты за изоляторы. |
| **Recall** | > 0.88 | Модель находит подавляющее большинство реальных дефектов на снимках. |
| **mAP@50** | > 0.90 | Средняя точность детекции, подтверждающая надежность системы. |

*Модель демонстрирует стабильную работу при различных условиях освещенности и ракурсах съемки.*

## 5. Как работает система

Система состоит из двух логических блоков:

* **Ядро детекции (`processor.py`):** Загружает обученные веса (`best.pt`), принимает на вход изображения, выполняет поиск объектов и сохраняет результат с рамками.
* **Аналитический блок (`test.py`):** Выполняет пакетную обработку всех изображений из папки `input`, генерирует итоговый `report.csv` с информацией: какой класс найден и с какой вероятностью (`confidence`).

## 6. Результаты внедрения

* **Автоматизация:** Полный отказ от ручного просмотра неповрежденных участков.
* **Скорость:** Модель обрабатывает сотни изображений за считанные минуты.
* **Точность:** Фиксация каждого инцидента с указанием вероятности, что позволяет оператору быстро принимать решения.
